### Model Descriptions

This document defines the available models provided by the `gemini` provider: **Gemini 2.0 Flash-Lite (Small)**, **Gemini 2.0 Flash Thinking (Medium)**, and **Gemini 2.5 Pro (Large)**. It outlines their capabilities, limitations, and recommended use cases to guide an automated model selection process based on task complexity, performance needs, and inferred computational cost. The primary directive is effectiveness balanced with efficiency: **default to the Flash Thinking (Medium) model for general tasks, select Flash-Lite (Small) specifically for simple exploration or maximum efficiency on low-complexity tasks, and escalate to Pro (Large) only when necessary.**

All models generally support large context windows (up to 1 million tokens), though output limits and specific performance within large contexts vary. Computational cost increases significantly with model size and complexity tier.

**Core Selection Principle:** Prioritize **Flash Thinking (Medium)** as the default workhorse model. Select **Flash-Lite (Small)** explicitly when the task is clearly identified as simple code exploration, structural description, basic formatting, or when cost/speed are the absolute priority for a demonstrably low-complexity request. Escalate to **Pro (Large)** only for exceptionally complex tasks demonstrably exceeding Medium's capabilities.

---

#### Gemini 2.0 Flash-Lite (Small Model) - *For Exploration & Efficiency*

```json
{
  "provider": "gemini",
  "model": "small"
}
```

**Provider**: gemini
**Model Identifier**: `gemini-2.0-flash-lite`

**Description**:
The most lightweight, fastest, and most cost-efficient model. Optimized for speed, low latency, and low resource usage, it excels at simple code exploration, structural analysis, and basic tasks where efficiency is paramount. Likely supports a 1M token context window, optimized for efficient processing of straightforward requests.

**Strengths**:
- **Lowest Cost & Highest Speed**: Most efficient in terms of resource usage and latency, ideal for high-volume, low-complexity tasks or basic code lookups.
- **Code Exploration**: Highly effective and efficient for searching codebases, locating implementations, and describing basic code structure.
- **Efficient Large Context (Simple Tasks)**: Suitable for quickly searching or retrieving specific information from large inputs when deep cross-context reasoning isn't required.
- **Basic Task Handling**: Capable performance on straightforward classifications, simple formatting, and basic autocompletion. Achieves a Chatbot Arena coding score of 1318.

**Use Cases**: **(Select specifically for these tasks)**
- Searching codebases ("Where is X implemented?", "Show me the definition of Y").
- Describing basic code structure or function signatures.
- Simple code formatting.
- Classification or categorization based on clear patterns.
- Autocompletion suggestions.
- Simple data extraction based on explicit patterns.
- Request routing based on keywords or simple intent.

**Limitations**:
- Not suited for tasks requiring diagnostic reasoning, analysis of interactions, understanding complex logic, or nuanced interpretation.
- Struggles with multi-step planning, moderately complex code generation, or debugging beyond simple syntax errors.
- Performance significantly lower than Medium/Large on complex coding benchmarks.

**Selection Guidance**:
**Do not select by default.** Select **Flash-Lite (Small)** *explicitly* when the incoming query is clearly identified as a simple code exploration task (e.g., finding definitions, basic structure description), requires simple formatting, or when maximum speed/lowest cost is the primary driver for a demonstrably low-complexity task. **For most coding assistance, generation, debugging, and analysis tasks, default to Flash Thinking (Medium).**

#### Gemini 2.0 Flash Thinking (Medium Model) - *Default Choice*

```json
{
  "provider": "gemini",
  "model": "medium"
}
```

**Provider**: gemini
**Model Identifier**: `gemini-2.0-flash-thinking-exp-01-21`

**Description**:
A highly capable model optimized for speed *and* reasoning, offering a strong balance of analytical and coding abilities suitable for a wide range of tasks. **This is the recommended default model for general coding assistance, debugging, and analysis.** It handles analysis of interactions, moderately complex code generation, and multi-step instructions effectively. Features a 1M token context window and an 8,192 token output limit.

**Strengths**:
- **Balanced Performance**: Offers a strong blend of reasoning, coding capability, and speed, making it suitable for interactive use.
- **Good Coding/Debugging**: Proficient at identifying bugs, understanding interaction effects, generating/refactoring moderately complex code, and explaining logic. Achieves a Chatbot Arena coding score of 1365.
- **Interaction Analysis**: Capable of analyzing how different code components interact and diagnosing moderately complex issues.
- **Complex Instruction Following**: Reliable for handling intricate, multi-part instructions.
- **Effective Large Context Analysis**: Can reason efficiently across large contexts for tasks requiring synthesis or understanding beyond simple lookups.

**Use Cases**: **(Default selection for most tasks)**
- General code generation (functions, classes, snippets).
- Debugging moderately complex issues, including interaction bugs ("Why is X failing when Y calls it?").
- Explaining code logic and algorithms.
- Planning implementation approaches for features.
- Analyzing system behavior and interaction effects.
- Refactoring code with moderate dependencies.
- Answering questions requiring reasoning across different parts of the codebase.
- Assisting with understanding and navigating moderately complex codebases.

**Limitations**:
- Higher computational cost and potentially higher latency compared to **Flash-Lite (Small)**.
- Smaller output token limit (8k) compared to **Pro (Large)** (65k).
- May not handle the absolute most complex architectural planning or debugging scenarios as effectively as **Pro (Large)**.

**Selection Guidance**:
**Select Flash Thinking (Medium) by default for most incoming coding-related queries.** Use it for general code assistance, generation, debugging, analysis of interactions, and moderately complex planning. **Downgrade to Flash-Lite (Small) *only if* the task is explicitly identified as simple exploration/description or requires maximum efficiency for a trivial request.** Escalate to **Pro (Large)** *only when* the task complexity (e.g., large-scale architectural design, deep systemic bug analysis) clearly exceeds the capabilities of **Flash Thinking (Medium)**.

#### Gemini 2.5 Pro (Large Model) - *Exceptional Use Only*

```json
{
  "provider": "gemini",
  "model": "large"
}
```

**Provider**: gemini
**Model Identifier**: `gemini-2.5-pro-preview-03-25`

**Description**:
Google's most advanced and capable reasoning model, designed for the most complex tasks requiring deep understanding, planning, and state-of-the-art coding abilities. Features built-in thinking capabilities, a 1M token context window, and a large 65,536 token output limit. It represents the **highest tier of computational cost and capability**. Its selection should be triggered **extremely rarely**.

**Strengths**:
- **Peak Performance & Reasoning**: Highest scores across coding benchmarks and reasoning tasks.
- **Deepest Strategic & Coding Ability**: Unmatched for high-level architectural design, complex system analysis, strategic planning, and tackling difficult coding challenges.
- **Large Output Capacity**: Can generate extensive code, detailed explanations, and complex documents up to 65k tokens.
- **Superior Complex Problem Solving**: Excels at tasks with high ambiguity, requiring synthesis across vast amounts of information or novel problem-solving approaches.

**Use Cases**: **(Select ONLY in extremely rare cases of highest complexity)**
- Generating foundational system architecture designs for novel, large-scale, or highly complex systems.
- Devising strategic plans for major, cross-cutting refactoring efforts across vast or intricate codebases.
- Performing deep analysis requiring synthesis of information across extremely large, complex codebases or datasets where **Flash Thinking (Medium)** is predicted to be insufficient.
- Addressing tasks equivalent in complexity to principal-level architectural design, advanced research problems, or debugging highly elusive, systemic issues.
- Generating very large, complex code structures or detailed technical documentation.

**Limitations**:
- **Highest Computational Cost & Latency**: Resource-intensive and slower compared to **Flash-Lite (Small)** / **Flash Thinking (Medium)**.
- **Massive Overkill**: Grossly inefficient and unnecessarily expensive for anything other than its narrow band of intended high-complexity tasks.

**Selection Guidance**:
**Select Pro (Large) *only as a final resort* when task analysis indicates complexity corresponding to foundational architectural design, highly complex multi-system strategic planning, deep synthesis requirements across massive contexts, or coding/debugging challenges demonstrably beyond Flash Thinking's (Medium) capabilities.** This selection signifies the detection of the absolute highest level of task complexity, ambiguity, and reasoning depth, where the default **Flash Thinking (Medium)** model is predicted to fail comprehensively. Its selection should be an indicator of an exceptional query requiring maximum computational effort and reasoning power.
