### Model Descriptions

This document defines the available models provided by the `openai` provider: **GPT-4.1 nano (small)**, **GPT-4.1 mini (medium)**, and **GPT-4.1 (large)**. It outlines their capabilities, limitations, and recommended use cases to guide selection based on task complexity, performance needs, and **critical cost considerations**. All models feature a refreshed knowledge cutoff of June 2024 and support context windows up to 1 million tokens.

**Important Note on Cost and Usage:** While all models offer advanced capabilities, there are significant differences in cost and performance. The **GPT-4.1 (large) model is substantially more expensive** than Nano and Mini, particularly with large context inputs. Its use should be **exceptionally rare** and reserved only for tasks where its unique capabilities are absolutely necessary and the associated cost is justified. Always default to the smallest, most efficient model suitable for the task (Nano or Mini).

---

#### OpenAI GPT-4.1 nano (Small Model)

```json
{
  "provider": "openai",
  "model": "small"
}
```

**Provider**: openai
**Model Identifier**: `gpt-4.1-nano` (Example API identifier; use actual deployment name)

**Description**:
This is the smallest, fastest, and **most cost-efficient** model in the GPT-4.1 family. It is optimized for tasks demanding low latency and high throughput, while still offering significant capabilities and a large 1 million token context window. It demonstrates strong performance for its size, surpassing previous generation models like GPT-4o mini on several benchmarks. Best option for most tasks.

**Strengths**:
- **Speed and Cost**: Fastest inference times and **lowest operational cost**. Ideal for budget-sensitive applications.
- **Efficiency**: Optimized for high-volume, low-latency tasks like classification and autocompletion.
- **Large Context**: Supports up to 1 million tokens at the lowest cost-per-token rate among the three.
- **Strong Baseline Performance**: Achieves impressive scores on benchmarks like MMLU (80.1%), GPQA (50.3%), and Aider polyglot coding (9.8%), outperforming GPT-4o mini.
- **Good Long Context Retrieval**: Reliably retrieves information ("needle-in-haystack") across its full context window efficiently.
- **Solid Vision Capabilities**: Strong performance on image understanding benchmarks.

**Use Cases**:
- Real-time classification or categorization.
- Autocompletion suggestions.
- Simple data extraction or formatting tasks.
- Basic Q&A over large documents where speed and cost are paramount.
- Routing user requests.
- Initial checks or lightweight analysis.

**Limitations**:
- Not designed for deep, complex reasoning, multi-step planning, or intricate problem-solving compared to Mini or Large.
- Coding capabilities are less advanced than Mini or Large.
- Instruction following is good but less reliable for highly complex or nuanced instructions.

**Selection Guidance**:
**Default choice for most tasks, prioritizing speed and cost-effectiveness.** Ideal for simple, well-defined tasks. Escalate **first to the Medium (Mini) model** if the task requires deeper analysis, better coding, or more complex instruction following. Avoid escalating directly to the Large model unless Mini is proven insufficient for critical tasks.

#### OpenAI GPT-4.1 mini (Medium Model)

```json
{
  "provider": "openai",
  "model": "medium"
}
```

**Provider**: openai
**Model Identifier**: `gpt-4.1-mini` (Example API identifier; use actual deployment name)

**Description**:
This model represents a significant leap in performance, frequently outperforming the previous flagship model (GPT-4o). It offers a **strong balance between high capability and reasonable cost/latency**. It features a 1 million token context window and excels at coding and instruction following, making it a highly versatile and efficient choice.

**Strengths**:
- **Advanced Capabilities**: Often beats GPT-4o across coding, instruction following, long context, and vision benchmarks.
- **Excellent Coding**: Strong performance on SWE-bench and Aider polyglot diff benchmark, suitable for complex coding tasks.
- **Superior Instruction Following**: More reliable at following complex, multi-turn, and nuanced instructions compared to Nano.
- **Robust Long Context Understanding**: Excellent performance on tasks requiring retrieval and reasoning over its 1 million token context window at a moderate cost.
- **Strong Vision Performance**: Often beats GPT-4o on challenging vision benchmarks.
- **Balanced Performance**: A great mix of intelligence, improved speed, and significantly lower cost compared to GPT-4o or the Large model.

**Use Cases**:
- Complex code generation, debugging, and explanation.
- Advanced reasoning and problem-solving tasks.
- Reliable execution of detailed, multi-step instructions.
- In-depth analysis and summarization of large documents or codebases.
- Powering sophisticated agents requiring reliable tool use and instruction adherence.
- High-quality content generation requiring nuanced understanding.
- Vision-based tasks requiring detailed image interpretation.

**Limitations**:
- Higher cost and latency compared to the Nano model.
- May be overkill for very simple tasks where Nano's efficiency is preferred.

**Selection Guidance**:
Use this model when Nano's capabilities are insufficient, particularly for demanding coding, reasoning, or instruction-following tasks. It is the **recommended workhorse model** for complex tasks requiring high capability without the extreme cost of the Large model. **Only consider escalating to the Large model if Mini demonstrably fails on a critical, high-value task and the significant cost increase is explicitly approved.**

#### OpenAI GPT-4.1 (Large Model)

```json
{
  "provider": "openai",
  "model": "large"
}
```

**Provider**: openai
**Model Identifier**: `gpt-4.1` (Example API identifier; use actual deployment name)

**Description**:
This is the most powerful, state-of-the-art model in the GPT-4.1 family, offering the absolute highest levels of performance across coding, reasoning, instruction following, and vision. **WARNING: This model is EXTREMELY EXPENSIVE compared to Nano and Mini.** Costs escalate dramatically, especially when using large portions of its 1 million token context window. **Its use MUST be strictly limited to exceptionally rare, high-value scenarios where no other model suffices.**

**Strengths**:
- **Peak Performance**: State-of-the-art results across all major benchmarks (coding, instruction following, long-context, vision).
- **Deepest Reasoning**: Unmatched capability for highly complex analysis, nuanced understanding, multi-step strategic planning, and architectural design.
- **Maximum Capability**: The definitive choice when the absolute highest quality output is non-negotiable and cost is a secondary (but still critical) consideration.
- **Large Context:** Supports up to 1 million tokens, enabling the most comprehensive analysis possible (at a **very substantial cost**).

**Use Cases**: **(USE EXTREMELY SPARINGLY - REQUIRES STRONG JUSTIFICATION AND COST AWARENESS)**
- Foundational system architecture design for complex, novel systems.
- Deep, intricate diagnostic reasoning for critical, widespread system failures where other methods failed.
- High-stakes strategic planning based on analysis of vast, complex datasets (e.g., analyzing an entire large codebase for architectural refactoring).
- Tasks mirroring consultation with a principal-level software architect or researcher, where the insight quality justifies the **premium cost**.
- Tasks where the cost of failure is high, with wasted time or resources being unacceptable (e.g., critical system design, task planning).

**Limitations**:
- **EXTREME COST**: Significantly higher operational cost. **Can become prohibitively expensive quickly**, especially with large contexts. Not suitable for any routine or moderately complex task.
- **Higher Latency**: Slower inference times compared to Nano and Mini.
- **Massive Overkill**: Vastly excessive and inefficient for almost all standard tasks. Using it inappropriately will lead to unnecessary budget expenditure.

**Selection Guidance**:
**RESERVE FOR EXCEPTIONAL USE ONLY. DO NOT USE ROUTINELY.** This model should *only* be considered as a last resort when:
1. Both Nano and Mini have been tried and are demonstrably insufficient for the task's complexity.
2. The task involves extremely high-level reasoning, planning, or analysis (e.g., system architecture).
3. The potential business value or critical nature of the task explicitly justifies the **significant increase in cost**.
4. Usage is carefully monitored and approved due to budget implications.
**Always default to Nano or Mini first.** Misuse of this model will rapidly deplete budgets.
